# Notes

Our best configuration uses a custom 2,048 vocabulary BPE tokenizer, weight tying, RMSNorm, SwiGLU MLP, and Rotary Position Embeddings (RoPE), running under a tuned 4-layer, 6-head, 180-embedding dimension configuration (1,928,340 parameters). The BPE tokenizer uses a cluster-aware regex splitter (`[\p{L}\p{M}]+`) that prevents splitting Devanagari consonants from their combining marks, achieving an efficient compression ratio of 3.34 bytes per token on the mixed corpus. Strict byte fallback maps unseen characters to base bytes U+0000–U+00FF, ensuring the model never crashes on unseen text. Tying the token embedding and linear output head weights eliminates redundant parameters, allowing us to allocate more capacity to representation learning. Replacing LayerNorm with RMSNorm improves CPU training speed and stability by omitting the mean computation. SwiGLU activations introduce multiplicative gating in the feedforward network, which significantly speeds up convergence. Rotary Position Embeddings (RoPE) provide relative position information directly inside SelfAttention, enabling better length generalization. Decoupled weight decay via AdamW (excluding biases/norms) prevents overfitting in the 2,000-step training regime. A cosine schedule with a 100-step linear warmup ensures smooth optimization steps and steady descent. Finally, gradient accumulation handles micro-batches of size 1 to enable an effective batch size of 256 tokens without exceeding CPU memory.
